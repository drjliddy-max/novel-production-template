# Authoring Voice Categories

This file defines the voice categories used across John Liddy's authoring workflows.

## Nonfiction Book Voice

Canonical source:

- `style-guide/JOHN_LIDDY_VOICE.md`

Applies to:

- The Participation Effect
- future nonfiction books
- nonfiction essays
- book-adjacent articles
- emails and landing copy written in John's voice

Core rule:

Sound like John, not AI. Direct, specific, lived-in, conversational, and precise.

Required traits:

- short paragraphs
- varied sentence length
- open with the idea
- no em dashes
- no semicolons
- no ellipses
- no formulaic transitions
- specific vocabulary
- negate before revealing where it earns its place
- stories prove concepts, then move on

## Fiction Book Voice

Canonical source:

- each fiction repo's `CLAUDE.md`
- for Quatora, `/Users/johnliddy/Desktop/Projects/the-war-for-paratua/CLAUDE.md`

Applies to:

- The War for Paratua
- The Broken Crown
- future fiction set in a project-specific world

Core rule:

Preserve canon first. Then write in the project-specific narrative voice.

For Quatora, the voice is epic fantasy with Tolkien, Donaldson, and Chalker influences. It is intentionally different from John's nonfiction voice.

## Blog Voice

Canonical source:

- Site Monitor project voice files at `projects/<siteKey>/voice.md`
- each voice file may reference `JOHN_LIDDY_VOICE.md`, repo-specific `CLAUDE.md`, schedule rules, and live published examples

Core rule:

Every blog post must read the correct site-specific writer reference before drafting. The voice file must include:

- Voice
- Stats
- Stories
- Opinions
- Humor
- Safety

## Other Writing

For outreach, hero copy, book descriptions, product copy, and public-facing emails, use `JOHN_LIDDY_VOICE.md` unless a repo-specific `CLAUDE.md` or project voice file narrows the category.

The category-specific file can narrow the voice. It cannot make the writing generic.
